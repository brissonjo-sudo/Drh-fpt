# Gabarits de livrables — assets/

Modèles réutilisables de livrables RH, **classés par niveau** (le niveau guide
le format et les mentions obligatoires).

## Classement

1. **Décision** — arrêté individuel, décision individuelle.
   Nommage : `decision-objet.md` (ex. `decision-mise-en-disponibilite.md`).
2. **Organisation** — procédure, fiche, mode opératoire.
   Nommage : `procedure-objet.md`.
3. **Pilotage** — note d'aide à la décision, tableau de bord, délibération.
   Nommage : `note-objet.md`, `deliberation-objet.md`.
4. **Communication** — courrier d'agent, note de service, FAQ.
   Nommage : `courrier-objet.md`.

## Éléments obligatoires par type

- **Tout acte faisant grief** (refus, sanction, décision défavorable) :
  visa des textes, **motivation en fait et en droit**, **voies et délais de
  recours** (gracieux, hiérarchique, contentieux devant le tribunal
  administratif), signature de l'autorité, et vérification de l'obligation de
  **transmission au contrôle de légalité**.
- **Délibération** : visa, exposé des motifs, dispositif, mention de l'avis
  préalable requis le cas échéant (CST pour les LDG, par exemple).
- **Courrier d'agent** : objet clair, réponse motivée, voie de recours si refus.

## Conventions

- Champs à compléter entre crochets `[ ]`.
- Points à vérifier juridiquement signalés explicitement avant signature.
- **Aucune donnée nominative** dans les gabarits.

## Fiche profil

`fiche-profil-collectivite.md` — gabarit du cadrage d'ouverture (SKILL.md §6.1).
Établit le profil de la collectivité (type, statut particulier, affiliation,
filières) pour calibrer les réponses et cibler les recherches. Réutilisable via
trois modes de persistance (recoller / mémoire Claude / fichier local).

## Enrichissement

Le dossier démarre léger. Quand un même livrable est demandé plusieurs fois, le
skill **propose de créer le gabarit** correspondant ici.
